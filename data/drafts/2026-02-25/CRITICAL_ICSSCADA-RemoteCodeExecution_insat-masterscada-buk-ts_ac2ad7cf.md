
# InSAT MasterSCADA BUK-TS

> ### Executive Overview
러시아 기반의 InSAT사가 개발한 MasterSCADA BUK-TS 솔루션에서 원격 코드 실행(RCE)이 가능한 두 개의 치명적인 취약점이 발견되었습니다. 이 시스템은 제조, 에너지, 수처리 등 전 세계 핵심 인프라 분야에서 광범위하게 사용되고 있어, 공격 성공 시 국가 기반 시설의 중단이나 물리적 피해를 야기할 수 있는 심각한 위협으로 간주됩니다. 특히 제조사가 CISA의 보안 권고 및 완화 조치 협력 요청에 응답하지 않고 있어, 현재 공식적인 패치가 존재하지 않는 제로데이(Zero-day) 상태에 가깝습니다.

### Threat Analysis
이번에 식별된 위협은 CVE-2026-21410(SQL 인젝션)과 CVE-2026-22553(OS 커맨드 인젝션)으로 구성됩니다. CVE-2026-21410은 메인 웹 인터페이스의 취약한 엔드포인트를 통해 악의적인 SQL 명령을 주입함으로써 데이터베이스 탈취 및 RCE를 허용합니다. CVE-2026-22553은 MMadmServ 웹 인터페이스의 특정 필드에서 입력값 검증 미흡으로 발생하며, 공격자가 기본 운영 체제에서 직접 명령을 실행할 수 있게 합니다. 두 취약점 모두 CVSS 9.8의 높은 점수를 기록하며, 특권 권한 없이도 네트워크를 통해 공격이 가능합니다.

### Impact Assessment
본 취약점이 악용될 경우, 공격자는 SCADA 서버에 대한 완전한 제어권을 획득할 수 있습니다. 이는 단순히 데이터 유출에 그치지 않고, 산업 제어 시스템(ICS)을 조작하여 생산 라인 가동 중단, 전력망 차단, 또는 수처리 시설의 물리적 오작동을 유발할 수 있습니다. 전 세계적으로 배치된 장비 특성상 공급망 공격의 교두보가 될 위험이 크며, 특히 패치 부재로 인해 위협 노출 기간이 장기화될 것으로 우려됩니다.

### Recommendations
공식 패치가 배포될 때까지 조직은 즉각적인 방어 조치를 시행해야 합니다. 첫째, 모든 제어 시스템 장비를 인터넷에서 차단하고 네트워크 분리(Air-gapping)를 강화하십시오. 둘째, 원격 접속이 불가피한 경우 최신 버전의 VPN을 사용하되, 비즈니스 네트워크와 ICS 네트워크 사이에 엄격한 방화벽 정책을 적용하십시오. 셋째, MITRE ATT&CK T1190 기법에 대응하기 위해 웹 인터페이스 유입 트래픽에 대한 심층 패킷 분석(DPI) 및 침입 탐지 시스템(IDS) 룰을 업데이트할 것을 권고합니다.

**Tags**: ICS/SCADA, Remote Code Execution, SQL Injection, OS Command Injection, Critical Infrastructure
